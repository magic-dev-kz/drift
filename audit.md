Качественная реализация. Один HTML-файл, чистый код, точное следование дизайн-системе. Все 10 AC выполнены. Есть несколько багов средней тяжести, связанных с accessibility и edge-кейсами.

## Что хорошо

- **localStorage обёрнут в try/catch** -- loadData(), saveData(), clearData() все три в try/catch. AC2 выполнено образцово.
- **Momentum = скользящие 30 дней** -- calcMomentum() считает процент из последних 30 дней, не streak. Формула точно соответствует spec.
- **Запрещённые слова** -- ни "стрик", ни "streak", ни "серия", ни "подряд" не встречаются ни в UI, ни в JS-коде. AC8 выполнено.
- **Фразы позитивные** -- все 6 фраз из spec присутствуют, ни одна не стыдит. Даже 0% -- "Первый шаг -- самый важный".
- **prefers-reduced-motion** -- полностью реализовано: все анимации/транзишены сброшены, частицы скрыты. Соответствует design.md секция 9.
- **Мобильный viewport** -- meta viewport есть, три брейкпоинта (<375, 375-767, 768+), flex-wrap на тепловой карте. AC9 выполнено.
- **Тёмная тема** -- полностью реализована через prefers-color-scheme, хотя spec пометил её "вне скоупа, если не бесплатна". Бесплатна -- сделали. Хорошо.
- **Дизайн-система** -- палитра, типографика, размеры, скругления, тени, анимации -- всё точно по design.md. CSS-токены совпадают 1:1.
- **Undo** -- повторное нажатие "Готово!" снимает отметку. AC4 выполнено.
- **Сброс привычки** -- долгий тап на названии -> модалка с подтверждением. AC10 выполнено.
- **Particle burst** -- 8 частиц, радиальное разлетание, цвет = текущий momentum. Соответствует design.md.
- **Haptic feedback** -- navigator.vibrate(10) в try/catch.
- **Одна привычка** -- нельзя добавить вторую, пока не удалишь текущую. AC3 выполнено.

## Баги

### BUG-1: Focus trap в модалке отсутствует (Severity: Medium)

**Где:** Модалка сброса привычки (modal-overlay).

**Проблема:** При открытой модалке Tab уходит на элементы под оверлеем (кнопка "Сделано", input названия). Пользователь с клавиатуры может взаимодействовать с интерфейсом за модалкой. Это нарушение WCAG 2.4.3 (Focus Order) и ожидаемого поведения role="dialog".

**Исправление:** Добавить focus trap: при открытии модалки перехватывать Tab/Shift+Tab и зацикливать фокус между "Отмена" и "Удалить". При закрытии -- возвращать фокус на элемент, вызвавший модалку. Также добавить закрытие по Escape.

---

### BUG-2: Модалка не закрывается по Escape (Severity: Medium)

**Где:** modal-overlay.

**Проблема:** Нет обработчика keydown для Escape. Пользователь с клавиатуры не может закрыть модалку стандартным способом.

**Исправление:** Добавить `document.addEventListener('keydown', ...)`, проверять `e.key === 'Escape'`, вызывать hideModal().

---

### BUG-3: Кнопки модалки не имеют focus-visible стилей (Severity: Low)

**Где:** .modal-btn (Отмена / Удалить).

**Проблема:** Нет стиля :focus-visible для кнопок модалки. При навигации Tab-ом пользователь не видит, какая кнопка в фокусе. Основные кнопки (.btn-done, .onboarding-btn) имеют focus-visible, а модальные -- нет.

**Исправление:** Добавить `.modal-btn:focus-visible { outline: 2px solid var(--color-accent-lavender); outline-offset: 3px; }`.

---

### BUG-4: WCAG AA контраст -- phrase text в светлой теме на грани (Severity: Low)

**Где:** Позитивная фраза: `--color-text-phrase: #7B6F65` на `--color-surface: #FFFFFF`.

**Проблема:** Контраст #7B6F65 на #FFFFFF = ~4.56:1. Для обычного текста 16px WCAG AA требует >= 4.5:1. Формально проходит, но впритык. При subpixel rendering или на дешёвых экранах может быть на грани читаемости.

**Рекомендация:** Не блокер, но стоит затемнить на 1-2 шага (например #6F6358) для запаса.

---

### BUG-5: Тепловая карта не имеет aria-label / accessibility описания (Severity: Low)

**Где:** div#heatmap.

**Проблема:** 30 точек без текстовых меток. Скринридер прочитает пустые div-ы. Momentum meter тоже не имеет aria-label -- процент виден визуально, но не объявлен семантически.

**Исправление:** Добавить `aria-label="Momentum: XX%"` на .momentum-meter. Добавить `role="img" aria-label="Отмечено X из 30 дней"` на .heatmap.

---

### BUG-6: Onboarding input -- нет focus-visible кастомного стиля (Severity: Low)

**Где:** .onboarding-input:focus.

**Проблема:** `outline: none` убирает стандартный фокус-индикатор. Замена -- подсветка border-color на лавандовый. Визуально работает, но пользователь с клавиатуры может не заметить только смену цвета рамки. Рекомендуется оставить или добавить outline для :focus-visible.

---

### BUG-7: Habit name input -- outline: none без адекватной замены (Severity: Low)

**Где:** .habit-name:focus.

**Проблема:** Аналогично BUG-6. При фокусе убирается outline, замена -- фоновая подсветка (background: var(--color-dot-missed)). На маленьких экранах может быть едва заметна. Нет отдельного :focus-visible стиля.

**Исправление:** Добавить `.habit-name:focus-visible { outline: 2px solid var(--color-accent-lavender); outline-offset: 2px; }`.

---

### BUG-8: Long press на мобильном может конфликтовать с выделением текста (Severity: Low)

**Где:** Обработчик pointerdown на habit-name input.

**Проблема:** На мобильных устройствах долгий тап на текстовом input-е обычно вызывает контекстное меню выделения текста. Код очищает longPressTimer при contextmenu, но не предотвращает его. Это значит, что пользователь увидит и контекстное меню, и модалку одновременно в некоторых ситуациях (timing race). Кроме того, если пользователь хочет выделить текст для редактирования, ему сложно не триггернуть модалку.

**Рекомендация:** Рассмотреть альтернативный триггер для сброса (например, отдельная иконка/кнопка внизу карточки с `font-size: small` и неброским стилем).
